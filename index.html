📌 ¿Qué es la Programación Orientada a Objetos?

La programación orientada a objetos (POO) es un paradigma de programación que organiza el software en torno a objetos, en lugar de acciones, y datos en lugar de lógica.

¿Qué es un objeto?
Un objeto es una unidad que combina datos y comportamiento. Es como una entidad del mundo real: tiene atributos (características) y métodos (acciones).

Por ejemplo, si estás programando un videojuego con un personaje:


class Personaje {
  String nombre;
  int vida;

  void atacar() {
    print("$nombre ataca");
  }
}

Aquí, Personaje es una clase (el molde o plantilla), y cada personaje del juego es un objeto creado a partir de esa clase.

🔹 Conceptos Clave de la POO

    🧱 1. Encapsulamiento El encapsulamiento es el principio de ocultar los datos internos de un objeto y restringir el acceso directo a ellos. En su lugar, se accede a través de métodos públicos (getters y setters).

    📌 ¿Por qué es importante? Principalmente protege la integridad de los datos (evita modificaciones no controladas). Facilita el mantenimiento del código, ya que puedes cambiar la implementación interna sin afectar a quien usa la clase.

    🧬 2. Herencia La herencia permite crear nuevas clases (subclases) que heredan atributos y métodos de clases ya existentes (superclases). Esto permite reutilizar código y establecer jerarquías.

    📌 ¿Por qué es importante? Facilita la reutilización de código. Este tiene la Cualidad de interconectar, crear estructuras jerárquicas (por ejemplo, Animal > Perro > Pastor Alemán). Promueve el diseño DRY (Don't Repeat Yourself).

    🧩 3. Polimorfismo El polimorfismo significa que un mismo método puede tener diferentes comportamientos según el objeto que lo llame. Hay dos tipos:

    🔄 Tipos de Polimorfismo: Polimorfismo en tiempo de compilación (sobrecarga de métodos) Polimorfismo en tiempo de ejecución (sobrescritura de métodos)

    📌 ¿Por qué es importante? Permite escribir código flexible y extensible. Hace posible el uso de interfaces y clases abstractas. Permite tratar objetos diferentes como si fueran del mismo tipo base

    🧠 4. Abstracción La abstracción consiste en mostrar solo los detalles esenciales de un objeto y ocultar la complejidad interna. Se implementa con clases abstractas o interfaces.

    📌 ¿Por qué es importante? Permite enfocarse en qué hace un objeto, no cómo lo hace. Facilita el diseño de sistemas complejos de forma más clara y modular. Todos estos conceptos estaran siendo desarrollados en la pestaña de Principios
